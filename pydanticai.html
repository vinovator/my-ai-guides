Why PydanticAI? Basic Agent Dependency Injection Streaming Response

PydanticAI Refresher

Production-grade framework built for Type Safety and Reliability.

1. Foundations

Created by Samuel Colvin (creator of Pydantic). While other frameworks (LangChain) are "Playgrounds", PydanticAI is "Production Engineering". It focuses on Validation, Testing, and Dependency Injection.

2. The Agent

from pydantic_ai import Agent

# Define agent with model
agent = Agent(
    'openai:gpt-4o',
    system_prompt='Be helpful and concise.'
)

# Run Sync
result = agent.run_sync('What is the capital of France?')
print(result.data)

Technique: Dependency Injection

The "Killer Feature". Inject DB connections or API keys safely at runtime without global variables.

from dataclasses import dataclass
from pydantic_ai import RunContext

@dataclass
class MyDeps:
    db_connection_str: str
    user_id: int

agent = Agent('openai:gpt-4o', deps_type=MyDeps)

@agent.tool
async def get_user_data(ctx: RunContext[MyDeps]) -> str:
    # Use the injected dependency
    conn = ctx.deps.db_connection_str
    return f"Data for user {ctx.deps.user_id} from {conn}"

# Inject deps when running
deps = MyDeps(db_connection_str="postgres://...", user_id=123)
result = agent.run_sync('Who am I?', deps=deps)

Technique: Streaming

async with agent.run_stream('Tell me a story') as result:
    async for message in result.stream():
        print(message)
