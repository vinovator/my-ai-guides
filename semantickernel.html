Semantic Kernel (Microsoft)

Philosophy

Semantic Kernel (SK) is the Open Source SDK that powers Microsoft's Copilot stack. Unlike LangChain which tries to do everything, SK focuses on three clear concepts to connect LLMs to your code:

  • The Kernel: The central orchestrator.
  • Plugins: Your existing API/Code wrapped for the AI.
  • Planners: The AI logic that decides which plugin to call (the "Agent" part).

1. The Kernel

The kernel is the context holder. You attach AI services and Plugins to it.

import semantic_kernel as sk
from semantic_kernel.connectors.ai.open_ai import OpenAIChatCompletion

# 1. Initialize Kernel
kernel = sk.Kernel()

# 2. Add AI Service (OpenAI or Azure OpenAI)
service_id = "default"
kernel.add_service(
    OpenAIChatCompletion(
        service_id=service_id, 
        ai_model_id="gpt-4", 
        api_key="..."
    )
)

2. Plugins (The Core)

SK treats "Tools" as Plugins. A plugin is just a class with annotated functions.

from semantic_kernel.functions import kernel_function

class MathPlugin:
    @kernel_function(
        description="Adds two numbers",
        name="Add"
    )
    def add(self, number1: float, number2: float) -> float:
        return number1 + number2

# Import Plugin into Kernel
math_plugin = kernel.add_plugin(MathPlugin(), plugin_name="Math")

# Invoke directly (Manual)
result = await kernel.invoke(math_plugin["Add"], number1=5, number2=5)
print(result) # 10

3. Planners (The "Agent")

A Planner takes a user goal (e.g., "Calculate 5+5") and automatically constructs a plan using the available plugins.

from semantic_kernel.planners import FunctionCallingStepwisePlanner

# 1. Create Planner
planner = FunctionCallingStepwisePlanner(service_id=service_id)

# 2. Execute Goal
# The planner will see it has a 'Math' plugin and choose to use it.
result = await planner.invoke(kernel, "What is 10 plus 20?")

print(result.final_answer)
