Semantic Kernel (SK)

Microsoft's SDK for integrating LLMs with existing code. The engine behind "Copilot".

1. Foundations

SK is not just a chatbot library; it's a lightweight SDK to mix Native Code (C#, Python) with Semantic Functions (LLM Prompts). It is designed for Enterprise integration.

Why use SK? When you want to build a "Copilot" inside your own app that triggers your own APIs securely.

2. The Kernel

The central object. You attach "Services" (AI Models) and "Plugins" (Tools) to it.

import semantic_kernel as sk
from semantic_kernel.connectors.ai.open_ai import OpenAIChatCompletion

kernel = sk.Kernel()

# Add OpenAI Service
kernel.add_service(
    OpenAIChatCompletion(
        service_id="chat-gpt", 
        ai_model_id="gpt-4", 
        api_key="..."
    )
)

3. Plugins (Native Functions)

SK uses decorators to expose Python functions to the LLM.

from semantic_kernel.functions import kernel_function

class LightPlugin:
    @kernel_function(
        description="Turns the light on or off",
        name="ChangeState"
    )
    def change_state(self, state: bool) -> str:
        return f"Light is now {'on' if state else 'off'}"

# Add to Kernel
kernel.add_plugin(LightPlugin(), plugin_name="Lights")

Technique: Planners (The Agent)

A Planner uses the LLM to look at available plugins and solve a user goal automatically.

from semantic_kernel.planners import FunctionCallingStepwisePlanner

# 1. Initialize Planner
planner = FunctionCallingStepwisePlanner(service_id="chat-gpt")

# 2. Ask it to do something that requires the Plugin
goal = "Turn on the lights."
result = await planner.invoke(kernel, goal)

print(result.final_answer) 
# Output: "Light is now on" (After calling the python function)
