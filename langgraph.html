1. Foundations

What is it?

A library to build stateful, multi-actor applications with LLMs. It extends LangChain to handle cycles (loops), which are essential for agentic behaviors like "Plan -> Execute -> Reflect -> Retry".

When to use?

  • You need loops (retries, refinement).
  • You need persistence (pause/resume conversation).
  • You need fine-grained control over state.
  • Production-grade reliability is priority.

2. Core Theory: The State

LangGraph is distinct because it passes a shared State Object to every node. Nodes modify this state.

from typing import TypedDict, Annotated
import operator

# The Schema
class AgentState(TypedDict):
    # 'operator.add' means: when a node returns a message, APPEND it to this list
    # instead of overwriting the whole list.
    messages: Annotated[list, operator.add]
    current_step: str
    tool_output: dict

3. Building the Graph

from langgraph.graph import StateGraph, END

# 1. Define Nodes (Python Functions)
def chatbot(state: AgentState):
    return {"messages": ["Hello from AI"]}

def tool_node(state: AgentState):
    return {"messages": ["Tool Result"]}

# 2. Add to Graph
workflow = StateGraph(AgentState)
workflow.add_node("chatbot", chatbot)
workflow.add_node("tools", tool_node)

# 3. Define Flow (Edges)
workflow.set_entry_point("chatbot")
workflow.add_edge("chatbot", "tools")
workflow.add_edge("tools", END) # End of execution

# 4. Compile
app = workflow.compile()

Technique: Conditional Edges (Routing)

How agents make decisions (loops). "If tool called, go to Tool Node. Else, End."

def router(state: AgentState):
    last_msg = state["messages"][-1]
    if "tool_calls" in last_msg:
        return "tools" # Go to 'tools' node
    return "end"       # Go to END

workflow.add_conditional_edges(
    "chatbot", # From this node
    router,    # Run this logic
    {"tools": "tools", "end": END} # Map output to next node
)

Technique: Persistence (Checkpointing)

LangGraph can save the state after every step. This allows "Time Travel" and session resumption.

from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
app = workflow.compile(checkpointer=checkpointer)

# Run with a thread_id
config = {"configurable": {"thread_id": "session_1"}}
app.invoke({"messages": ["Hi"]}, config=config)

# Run again, it remembers "Hi"
app.invoke({"messages": ["My name is Bob"]}, config=config)

Technique: Human-in-the-loop

Pause execution before a specific node to let a human edit state or approve actions.

# Interrupt BEFORE entering the 'action' node
app = workflow.compile(
    checkpointer=checkpointer,
    interrupt_before=["action"] 
)

# The graph will run until it hits 'action', then pause.
# You can inspect state, update it, and then resume.
app.update_state(config, {"messages": ["Human override!"]})
app.invoke(None, config=config) # Resume
