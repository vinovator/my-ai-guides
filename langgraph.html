LangGraph Refresher

Stateful, Cyclic, Graph-based orchestration.

1. Foundations

A library to build stateful, multi-actor applications with LLMs. Unlike DAGs (CrewAI), LangGraph allows loops (cycles).

2. State & Schema

from typing import TypedDict, Annotated
import operator

class AgentState(TypedDict):
    # 'operator.add' appends messages instead of overwriting
    messages: Annotated[list, operator.add]
    current_step: str

3. Nodes & Edges

from langgraph.graph import StateGraph, END

def chatbot(state): return {"messages": ["Hello"]}

workflow = StateGraph(AgentState)
workflow.add_node("chatbot", chatbot)
workflow.set_entry_point("chatbot")
workflow.add_edge("chatbot", END)
app = workflow.compile()

Technique: Conditional Logic

Routing based on state content.

def route(state):
    return "tools" if "tool_calls" in state["messages"][-1] else "end"

workflow.add_conditional_edges("chatbot", route, {"tools": "tools", "end": END})

Technique: Persistence

from langgraph.checkpoint.memory import MemorySaver
app = workflow.compile(checkpointer=MemorySaver())
app.invoke(inputs, config={"configurable": {"thread_id": "1"}})

Technique: Human-in-the-loop

# Pause BEFORE entering 'action' node
app = workflow.compile(interrupt_before=["action"])

Technique: Subgraphs

You can treat an entire Graph as a Node in another Graph. Essential for scaling.

# Define a smaller graph
sub_builder = StateGraph(SubState)
sub_builder.add_node("worker", worker_node)
sub_graph = sub_builder.compile()

# Add it to parent graph as a node
parent_builder = StateGraph(ParentState)
parent_builder.add_node("subteam", sub_graph) # <--- Graph as a Node
