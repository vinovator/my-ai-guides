Core: The ReAct Agent

LlamaIndex excels at Data Agentsâ€”agents that can search your PDFs/SQL before answering. The most common pattern is ReAct (Reasoning + Acting).

from llama_index.core.agent import ReActAgent
from llama_index.llms.openai import OpenAI

# 1. Define LLM
llm = OpenAI(model="gpt-4")

# 2. Create Agent with Tools
agent = ReActAgent.from_tools(
    [multiply_tool, rag_tool], 
    llm=llm, 
    verbose=True
)

# 3. Chat
response = agent.chat("What is the revenue of Apple (from my docs) * 2?")

Turning Data into Tools

This is the superpower. You can turn an entire vector database search into a single "Tool" for the agent.

from llama_index.core.tools import QueryEngineTool, ToolMetadata

# Assume you have a 'query_engine' built from a PDF
query_engine_tool = QueryEngineTool(
    query_engine=query_engine,
    metadata=ToolMetadata(
        name="finance_docs",
        description="Provides financial info about the company."
    )
)

agent = ReActAgent.from_tools([query_engine_tool], llm=llm)

Workflows (New in v0.10)

Similar to LangGraph, LlamaIndex now supports event-driven workflows.

from llama_index.core.workflow import (
    StartEvent, StopEvent, Workflow, step
)

class MyFlow(Workflow):
    @step
    async def step_one(self, ev: StartEvent) -> StopEvent:
        return StopEvent(result="Hello World")

w = MyFlow(timeout=10, verbose=True)
result = await w.run()
