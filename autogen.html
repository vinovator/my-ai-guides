AutoGen Refresher

Microsoft's conversational framework. Best for code execution and dynamic agent teams.

1. Foundations

AutoGen defines an "Agent" as anything that can send/receive messages. The core loop is a Conversation. One agent speaks, another replies (either by executing code or using LLM), until a termination condition is met.

2. The Two Main Agents

AssistantAgent

The "Brain". Uses an LLM to generate plans, write code, or answer queries. It typically cannot execute code.

UserProxyAgent

The "Body". It acts on behalf of the user. It can execute Python/Bash code detected in the message, or ask the human for input.

from autogen import AssistantAgent, UserProxyAgent

config_list = [{"model": "gpt-4", "api_key": "..."}]

assistant = AssistantAgent(
    name="Coder",
    llm_config={"config_list": config_list}
)

user_proxy = UserProxyAgent(
    name="Executor",
    human_input_mode="NEVER", # Automated
    code_execution_config={
        "work_dir": "coding",
        "use_docker": False # Set True for safety
    }
)

# Start the conversation
user_proxy.initiate_chat(assistant, message="Plot NVDA vs TSLA stock YTD.")

Technique: Group Chat (Orchestrator)

For >2 agents, you need a GroupChatManager. It uses an LLM to decide who speaks next (Speaker Selection).

from autogen import GroupChat, GroupChatManager

# 1. Define Agents
coder = AssistantAgent(name="Coder", ...)
pm = AssistantAgent(name="PM", system_message="Critique plans.", ...)
reviewer = AssistantAgent(name="Reviewer", system_message="Check code.", ...)

# 2. Create Group
groupchat = GroupChat(
    agents=[user_proxy, coder, pm, reviewer], 
    messages=[], 
    max_round=10,
    speaker_selection_method="auto" # Or "round_robin"
)

# 3. Create Manager
manager = GroupChatManager(groupchat=groupchat, llm_config=...)

# 4. Start
user_proxy.initiate_chat(manager, message="Build a calculator.")

Technique: Registering Tools

AutoGen (v0.2+) supports OpenAI tool calling cleanly.

from autogen import register_function

def add(a: int, b: int) -> int:
    return a + b

# Registering connects the Python function to BOTH agents:
# 1. The caller (Assistant) gets the tool definition in its system prompt.
# 2. The executor (UserProxy) gets the ability to actually run the function.
register_function(
    add,
    caller=assistant,
    executor=user_proxy,
    name="add",
    description="Adds two integers."
)
