Philosophy

AutoGen abstracts multi-agent systems as conversations. Agents send messages to each other. Some agents are LLMs (Assistant), some are code executors (UserProxy), and some are humans.

1. Two Main Agent Types

AssistantAgent

The "AI". It proposes plans, writes code, and answers questions. It typically cannot run code itself.

UserProxyAgent

The "Executor". It acts as a proxy for the human. It can execute code detected in the Assistant's reply and send the result back.

from autogen import AssistantAgent, UserProxyAgent

config_list = [{"model": "gpt-4", "api_key": "..."}]

# The AI
assistant = AssistantAgent(
    name="Assistant",
    llm_config={"config_list": config_list}
)

# The Executor
user_proxy = UserProxyAgent(
    name="User_Proxy",
    code_execution_config={"work_dir": "coding", "use_docker": False},
    human_input_mode="TERMINATE" # Ask human only if termination msg not found
)

# Start Conversation
user_proxy.initiate_chat(
    assistant,
    message="Plot a chart of NVDA and TSLA stock price change YTD."
)

2. Group Chat (Orchestrator)

For more than 2 agents, you use a `GroupChat` and a `GroupChatManager` to decide who speaks next.

from autogen import GroupChat, GroupChatManager

coder = AssistantAgent(name="Coder", llm_config=llm_config)
pm = AssistantAgent(name="Product_Manager", system_message="Manage the plan.", llm_config=llm_config)
reviewer = AssistantAgent(name="Reviewer", system_message="Review code.", llm_config=llm_config)

# Define the Group
groupchat = GroupChat(
    agents=[user_proxy, coder, pm, reviewer], 
    messages=[], 
    max_round=12
)

# Define the Manager (The "Speaker Selection" logic)
manager = GroupChatManager(groupchat=groupchat, llm_config=llm_config)

# Start
user_proxy.initiate_chat(manager, message="Build a snake game in Python.")

3. Registering Tools

from autogen import register_function

def calculator(a: int, b: int) -> int:
    return a + b

# Register for both agents (One knows how to call, one knows how to execute)
register_function(
    calculator,
    caller=assistant,  # The LLM who will "call" it
    executor=user_proxy, # The local runtime that runs it
    name="calculator",
    description="Adds two numbers"
)
