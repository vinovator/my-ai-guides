CrewAI Refresher

The "Employee" model of Agentic AI. Focused on Role-Playing, Process, and Delegation.

1. Foundations

What is it?

An orchestration framework that treats agents as "employees" with distinct roles, goals, and backstories. It excels at automating known workflows (Sequential or Hierarchical) rather than open-ended conversations.

When to use CrewAI?

  • You have a known process (e.g., "Research -> Write -> Edit").
  • You want distinct "personas".
  • You need simple delegation without complex graph logic.

2. Agents & Roles

Agents are the "Doers". They wrap an LLM with specific instructions.

from crewai import Agent

researcher = Agent(
    role='Senior Researcher',
    goal='Uncover groundbreaking developments in {topic}',
    backstory="You are a veteran analyst who digs deep.",
    verbose=True,
    memory=True, # Enables Short/Long term memory
    tools=[]     # Optional list of tools
)

3. Tasks & Outputs

Tasks are the unit of work. They must have a clear description and an expected output format.

from crewai import Task

report_task = Task(
    description='Research {topic} and identify 5 key trends.',
    expected_output='A bulleted list of trends with sources.',
    agent=researcher,
    output_file='report.md', # Automatically saves output to file
    human_input=False,       # Set True to ask user for review
    async_execution=False    # Set True for parallel run
)

Technique: Hierarchical Process

A Manager Agent (LLM) plans the work and delegates it to workers automatically.

from crewai import Process, Crew
from langchain_openai import ChatOpenAI

manager_llm = ChatOpenAI(model="gpt-4")

crew = Crew(
    agents=[researcher, writer],
    tasks=[task1, task2],
    process=Process.hierarchical,        # <--- The Key Change
    manager_llm=manager_llm,             # <--- Mandatory for Hierarchical
    memory=True
)

Technique: Callbacks

Trigger functions when a task is completed.

def on_complete(output):
    print(f"Finished: {output.raw}")

task = Task(..., callback=on_complete)

Technique: CrewAI Flows (v0.80+)

Event-driven orchestration for chaining Crews.

from crewai.flow.flow import Flow, listen, start

class ContentPipeline(Flow):
    
    @start()
    def step_one(self):
        return "Topic"

    @listen(step_one)
    def step_two(self, topic):
        # Create a dynamic crew
        return Crew(...).kickoff(inputs={"topic": topic})

flow = ContentPipeline()
flow.kickoff()
