CrewAI Refresher

The "Employee" model of Agentic AI. Focused on Role-Playing, Process, and Delegation.

1. Foundations

What is it?

An orchestration framework that treats agents as "employees" with distinct roles, goals, and backstories. It excels at automating known workflows (Sequential or Hierarchical) rather than open-ended conversations.

History

Created by João Moura in late 2023. It quickly became the dominant Python framework for "Agentic Workflows" due to its simplicity compared to LangChain's verbose graph.

When to use CrewAI?

  • You have a known process (e.g., "Research -> Write -> Edit").
  • You want distinct "personas" (e.g., "The Critic", "The Creator").
  • You need simple delegation without complex graph logic.
  • You want built-in memory and RAG capabilities.

2. Core Theory: Agents & Tasks

A Crew is composed of Agents (the doers) and Tasks (the work). The magic happens in the Process.

basic_crew.py
from crewai import Agent, Task, Crew, Process

# 1. The Agent: A specific persona
researcher = Agent(
    role='Senior Researcher',
    goal='Uncover groundbreaking developments in {topic}',
    backstory="You are a veteran analyst who digs deep.",
    verbose=True,
    memory=True, # Enables Short/Long term memory
    tools=[]     # Optional list of tools
)

# 2. The Task: Specific unit of work
report_task = Task(
    description='Research {topic} and identify 5 key trends.',
    expected_output='A bulleted list of trends with sources.',
    agent=researcher,
    output_file='report.md' # Saves result automatically
)

# 3. The Crew: The container
crew = Crew(
    agents=[researcher],
    tasks=[report_task],
    process=Process.sequential # Default mode
)

Technique: Hierarchical Process

In this mode, you don't assign tasks to specific agents manually. A Manager Agent (usually GPT-4) plans the work and delegates it to the workers. 

from langchain_openai import ChatOpenAI

# The Manager needs a smart brain
manager_llm = ChatOpenAI(model="gpt-4")

crew = Crew(
    agents=[researcher, writer, editor],
    tasks=[loose_task_1, loose_task_2], # Tasks can be less specific about WHO does them
    process=Process.hierarchical,        # <--- The Key Change
    manager_llm=manager_llm,             # <--- Mandatory for Hierarchical
    respect_context_window=True,
    memory=True
)

Technique: Memory Systems

Why? Without memory, agents repeat work. CrewAI has 3 layers:
  • Short-term: Current execution context (RAG).
  • Long-term: Stored in local ChromaDB. Remembers past kickoffs.
  • Entity: Extracts and recalls specific nouns (Companies, People).
crew = Crew(
    agents=[...],
    tasks=[...],
    memory=True, # Enables all memory layers
    embedder={
        "provider": "openai",
        "config": {"model": 'text-embedding-3-small'}
    }
)

Technique: Callbacks & Human Input

How to intervene or trigger external API calls when a task finishes.

def on_task_complete(output):
    print(f"Task finished! sending email... Content: {output.raw[:50]}...")

task = Task(
    description='...',
    expected_output='...',
    agent=writer,
    human_input=True,     # Pauses for human approval before finalizing
    callback=on_task_complete # Triggers python function on success
)

Technique: CrewAI Flows (v0.80+)

Flows are event-driven. They allow you to chain multiple Crews together or run Python logic between Crews.

from crewai.flow.flow import Flow, listen, start

class ContentPipeline(Flow):
    
    @start()
    def generate_topics(self):
        return ["AI", "Cars"]

    @listen(generate_topics)
    def research_topic(self, topics):
        # Create a dynamic crew for the first topic
        crew = Crew(...)
        return crew.kickoff(inputs={"topic": topics[0]})

flow = ContentPipeline()
flow.kickoff()
