1 Foundational Concepts

What is CrewAI?

CrewAI is an open-source framework for orchestrating role-playing, autonomous AI agents. Unlike standard chatbots, CrewAI treats agents like employees in a company: each has a specific role, goal, backstory, and set of tools. They collaborate, delegate tasks, and share context to solve complex problems.

History & Origin

Created by João Moura in late 2023. It rapidly gained traction (becoming one of the fastest-growing GitHub repos) because it simplified the complexity of multi-agent systems into a clean, Pythonic API.

Core Philosophy

"Role-Playing" is the secret sauce. By assigning specific personas (e.g., "Senior QA Engineer" vs. "Junior Coder"), the LLM adopts specific behaviors, reducing hallucinations and improving focus.

Theoretical Concepts

Agents

Autonomous units that can perform tasks, make decisions, and communicate.

  • Role: What they do (e.g., "Analyst").
  • Goal: What they aim to achieve.
  • Backstory: Provides context/personality.
  • Tools: Capabilities (Search, Calc, etc.).

Tasks

Small, discrete assignments that agents execute.

  • Description: Clear instructions.
  • Expected Output: Definition of done.
  • Agent: Who is responsible (optional in hierarchical).

Process

How tasks are orchestrated.

  • Sequential: Task A -> Task B -> Task C.
  • Hierarchical: Manager assigns tasks dynamically.

Crew

The container layer.

  • Combines Agents + Tasks.
  • Defines the Process.
  • Handles Kickoff and Output.

Technique 1: Basic Sequential Crew

Foundational

The standard use case. Agents work in a relay race. The output of one agent becomes the context for the next.

main.py 
import os
from crewai import Agent, Task, Crew, Process

# 1. Define Agents
researcher = Agent(
    role='Senior Researcher',
    goal='Uncover groundbreaking technologies in {topic}',
    backstory="You are a curious mind, always exploring the latest developments.",
    verbose=True,
    allow_delegation=False,
    # llm=ChatOpenAI(model_name="gpt-4", temperature=0.7) # Optional custom LLM
)

writer = Agent(
    role='Tech Writer',
    goal='Narrate compelling tech stories about {topic}',
    backstory="You simplify complex topics into engaging narratives.",
    verbose=True,
    allow_delegation=False
)

# 2. Define Tasks
# Note: Tasks automatically pass context to the next task in Sequential mode
task1 = Task(
    description='Conduct a comprehensive analysis of {topic}. Identify key trends.',
    expected_output='Full analysis report in bullet points.',
    agent=researcher
)

task2 = Task(
    description='Using the insights provided, write an engaging blog post.',
    expected_output='A 4-paragraph blog post formatted in markdown.',
    agent=writer
)

# 3. Create Crew
crew = Crew(
    agents=[researcher, writer],
    tasks=[task1, task2],
    verbose=True,  # High-level logging
    process=Process.sequential  # Default mode
)

# 4. Kickoff
result = crew.kickoff(inputs={'topic': 'AI Agents in 2025'})
print("Final Output:", result)

Technique 2: Hierarchical & Delegation

Advanced

Concept: In this mode, you do not assign tasks to specific agents explicitly (or you can mix). Instead, a Manager Agent (created automatically or defined by you) acts as a boss. It breaks down the goal and delegates sub-tasks to the worker agents.

hierarchical_crew.py 
from crewai import Crew, Process, Agent, Task
from langchain_openai import ChatOpenAI

# Define a Manager LLM (The "Brain" of the operation)
# It needs to be a smarter model (e.g., GPT-4) to handle logic.
manager_llm = ChatOpenAI(model="gpt-4")

# Worker Agents (Note: allow_delegation=True is default, but good to be explicit)
analyst = Agent(
    role='Data Analyst',
    goal='Analyze data trends',
    backstory='Expert in reading spreadsheets and finding patterns.',
    allow_delegation=False
)

reporter = Agent(
    role='Reporter',
    goal='Report findings',
    backstory='Expert in communication.',
    allow_delegation=False
)

# Tasks - Notice we might be less prescriptive about WHO does it
# The manager will decide who is best suited or if both are needed.
project_task = Task(
    description="Analyze the Q3 market data and write a summary report.",
    expected_output="A summary report combined with data tables.",
    # No 'agent' assigned here! The manager will pick.
)

crew = Crew(
    agents=[analyst, reporter],
    tasks=[project_task],
    process=Process.hierarchical, # <--- The Magic Command
    manager_llm=manager_llm,      # <--- Mandatory for Hierarchical
    verbose=True
)

result = crew.kickoff()

Manual Delegation (Within Agents)

Even in Sequential processes, an agent can delegate if `allow_delegation=True`. The agent uses a built-in "Ask Co-worker" tool.

# Example of an agent allowed to delegate
senior_dev = Agent(
    role='Senior Developer',
    goal='Oversee code quality',
    backstory='You manage a team of juniors.',
    allow_delegation=True # <--- Gives them the "Delegate" tool
)

Technique 3: Memory Systems

CrewAI v0.30+ introduced sophisticated memory. Without this, agents have no context of past executions.

Short-Term

Context within the current run execution.

Long-Term

Stored in local DB (Chromadb). Remembers previous `kickoff()` results.

Entity Memory

Extracts and remembers key entities (Subject, People) discussed.

memory_config.py 
from crewai import Crew

# To enable memory, simply set memory=True.
# CrewAI handles the embeddings and storage automatically (usually in ./crews_data).

crew = Crew(
    agents=[...],
    tasks=[...],
    memory=True, # <--- Enables Short, Long, and Entity memory
    verbose=True,
    embedder={
        "provider": "openai",
        "config": {
            "model": 'text-embedding-3-small'
        }
    }
)

Technique 4: Custom Tools

Agents are only as good as their tools. There are two main ways to build them.

Method A: The Decorator (Simple)

from crewai_tools import tool

@tool("Stock Checker")
def check_stock(ticker: str):
    """Useful to check the stock price of a specific ticker."""
    # Logic here (e.g., API call)
    return f"The price of {ticker} is $150."

# Assign to agent
agent = Agent(..., tools=[check_stock])

Method B: Subclassing BaseTool (Robust)

from crewai_tools import BaseTool

class SentimentAnalysisTool(BaseTool):
    name: str = "Sentiment Analyzer"
    description: str = "Analyzes the sentiment of a text string."

    def _run(self, text: str) -> str:
        # Complex logic, maybe calling huggingface or another local model
        return "Positive"

sentiment_tool = SentimentAnalysisTool()

Technique 5: CrewAI Flows

New Feature (2025)

Flows are a higher-level abstraction introduced to manage event-driven workflows. Unlike a generic "Crew" which just runs tasks, a Flow can chain multiple Crews, run Python functions, and handle state more effectively.

flow_example.py 
from crewai.flow.flow import Flow, listen, start
from crewai import Crew, Agent, Task

class ContentFlow(Flow):
    
    @start()
    def generate_topics(self):
        # You can run raw python logic here, or a mini-crew
        print("Generating topics...")
        return ["AI", "Data", "Space"]

    @listen(generate_topics)
    def write_content(self, topics):
        # This function triggers when generate_topics finishes
        print(f"Received topics: {topics}")
        
        # We can dynamically create a crew here
        agent = Agent(role="Writer", goal="Write stuff", backstory="...")
        task = Task(description=f"Write a haiku about {topics[0]}", expected_output="Haiku", agent=agent)
        crew = Crew(agents=[agent], tasks=[task])
        
        return crew.kickoff()

flow = ContentFlow()
flow.kickoff()

Advanced Configurations

Structured Output (Pydantic)

Force an agent to return JSON/Pydantic objects instead of text strings.

from pydantic import BaseModel

class MovieReview(BaseModel):
    title: str
    rating: float
    summary: str

task = Task(
    ...,
    output_pydantic=MovieReview # Agent will strictly adhere to this schema
)

Human in the Loop

Stop execution and ask the human for feedback before finishing a task.

task = Task(
    description='Draft an email to the client.',
    expected_output='Email draft.',
    agent=writer,
    human_input=True # The script will pause and ask you for input/approval
)

Asynchronous Tasks

Run tasks in parallel if they don't depend on each other. 

task1 = Task(..., async_execution=True)
task2 = Task(..., async_execution=True)
# Both start immediately without waiting for the other
